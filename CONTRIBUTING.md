Thanks for wanting to contribute to NERAI!

Guidelines
- Please open an issue first to discuss larger changes or new features.
- Fork the repo and create feature branches (feature/..., fix...).
- Keep commits small and focused; use conventional commits when possible.
- Include tests for new features and ensure existing tests pass.

Pull request process
1. Fork and create a branch from main
2. Make changes and add tests
3. Run lint and tests locally
4. Submit a PR referencing the issue and include a clear description of changes

Code style
- TypeScript with strict mode enabled
- ESLint + Prettier (configured in repo)
